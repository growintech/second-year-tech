Think about it 🤔

Is this always exactly correct?

The calculation only uses the year. But what if your birthday hasn't happened yet this year?

If you were born in December and today is March, the formula would say you are already one year older than you actually are.

⭐
Bonus challenge — fix the edge case After calculating the year difference, check whether the birthday has already passed this year. If it hasn't, subtract 1. 
var now = new Date();
var birthday = new Date("2008-05-13");
var age = now.getFullYear() - birthday.getFullYear();

// Has the birthday already passed this year?
if (
  now.getMonth() < birthday.getMonth() ||
  (now.getMonth() == birthday.getMonth() &&
   now.getDate()  < birthday.getDate())
) {
  age = age - 1;
}

console.log("Exact age: " + age);
✅
Key idea Real-world programs need to handle these edge cases. Even a simple "how old am I?" feature is trickier than it looks!
