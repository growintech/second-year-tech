# **Replit AI**

**Lesson 16**

| Rationale In the previous lesson students experimented with an AI coding agent to generate a simple website. In this session they will use a more advanced tool: Replit. Replit is a development environment designed to build modern web applications. However, for this course students must still produce simple static websites using plain HTML, CSS and JavaScript. During the lesson students will learn how to guide the AI agent with clear prompts, inspect the generated project structure, and download the files to continue working locally. This activity also prepares students for the final project, where they will need to deliver a ZIP archive containing a static website. |      Goals Understand how a coding agent works inside a development environment Learn how to guide an AI tool using structured prompts Generate a simple static website project with AI Inspect and understand the generated code structure Download and reuse the generated files outside Replit Prepare for the final project requirements  |
| :---- | :---- |
| **Equipment and preparation** Computer class \+ computer \+ internet access \+ projector Presentation Ask students to create an account on **Replit**  | **Session duration** 1h30 |

# **Session brief**

| Subject | Activity | Notes | Time |
| :---: | ----- | :---: | ----- |
| **Coding Agent Recap** | Students already experimented with AI coding agents in the previous lesson. Quickly recap the idea: a coding agent is not just a chatbot that answers questions. It can also interact with a project, creating files, modifying code and updating the application automatically. However, the role of the developer does not disappear. Our job is still to: give clear instructions read the generated files check that the code is understandable improve the result step by step Explain that when using AI tools we should always think of them as assistants, not as replacements for developers. |  | 5 min |
| **Replit AI** | Introduce Replit as a **more complete development environment**, completely available online through a browser. Unlike the tool used in the previous lesson, Replit is designed to build quite complex web applications. For this reason, by default, it generates projects using modern frameworks or server technologies. Many of these technologies are **powerful** but **difficult to understand** for beginners. For this course we want to keep the generated projects **simple and readable**, using only: HTML CSS JavaScript Replit is still useful because it allows students to: generate the initial structure of a project inspect and modify files directly in the editor preview the result immediately download the code and continue working locally This last feature is very important because the final project must be delivered as a ZIP archive containing static files. Ask students to create a Replit account if they have not done it already. If possible, ask them to create the account before the lesson and not to experiment with the tool in advance, since the free version has a limited number of credits. |  | 10 min |
| **Initial Prompt** | Replit is a very modern tool and generates projects using technologies we have not studied yet. To ensure that the generated code matches our current knowledge, we must provide a **clear setup prompt before starting the project**. Create a simple static website project, a beginner-friendly static website that runs in Replit preview using a minimal server. `The website itself must use ONLY plain HTML, CSS and JavaScript. No frameworks. The website must have three files: index.html style.css script.js Put style and script in the /public folder. Create a minimal server so the project can run in Replit preview. The server must only serve the static files.`  Send this initial prompt to the students on Classroom. |  | 10 min |
| **Our Goal** | On the first lesson of April we will have the final test, an **individual test** that checks the knowledge of JavaScript concepts covered during the course. For this reason, students will use AI to generate a recap website that helps them review the main topics. The generated website could include, for example: a quiz about data types (numbers, strings, boolean…) a page explaining JSON a simple game about logical conditions small interactive exercises Encourage students to be creative, but remind them that the generated code must remain simple and understandable. |  | 30 min |
| **Inspect the code** | The first prompt might take some time to execute. Once the project is generated, explain the Replit interface: On the left side there is the **chat panel**, where students write instructions for the agent. In the center there is the **workspace**, where they can open files and see the project preview. Ask students to open the **file tree**. With the prompt provided earlier, the static files should be located in a folder similar to: `artifacts/<some-folder>/index.html`  Students should try to: open / edit the HTML file inspect the structure identify the CSS and JavaScript files Explain that sometimes it is faster to modify the code directly, instead of asking the agent to regenerate it. |  | 10 min |
| **Download the code** | Once students locate the folder containing the static files (HTML, CSS and JS), they can download the project. Steps: open the file explorer locate the folder containing the website files click the three dots select Download folder This downloads a ZIP archive. Ask students to: unzip the archive open index.html in the browser to try if the code works also locally. |  | 10 min |
| **The final project** | Explain that during the next weeks students will work on a final project, where they will build their own website using the knowledge acquired during the course. The project must be delivered as: a ZIP archive uploaded on Classroom, containing HTML, CSS and JavaScript files Students may use AI tools to help generate or modify code, but they must be able to: understand the generated structure explain the code during the presentation modify the project manually if needed More details about the final project in the dedicated files. |  | 15 min |
