The Initial Prompt

One specific prompt tells Replit to stay simple — and keeps the generated code readable.

Without guidance, Replit may generate a React app, a Node server with middleware, or other complex code. We can't understand that yet. The solution: send a precise setup prompt before anything else.

Each line of the prompt does a specific job:

📋 Initial Replit prompt — send this first
Create a simple static website project, a beginner-friendly static website that runs in Replit preview using a minimal server. goal
The website itself must use ONLY plain HTML, CSS and JavaScript. No frameworks. constraint
The website must have three files: index.html  style.css  script.js files
Put style and script in the /public folder. files
Create a minimal server so the project can run in Replit preview. The server must only serve the static files. server
💡
The server is only needed so Replit can preview the site in the browser. Your HTML/CSS/JS files stay completely static — no server code goes inside them.

Why each constraint matters

Without this line Replit may use React or Vue. Those frameworks add hundreds of files and concepts you haven't learned yet. Plain HTML/CSS/JS only.

Naming the files explicitly prevents Replit from inventing its own structure. You will always find index.html, style.css, and script.js — nothing unexpected.

Replit serves static files from a public-facing folder. Putting CSS and JS in /public means the browser can load them directly.

A tiny server (a few lines of Node.js) is enough to make Replit's preview work. It only serves the files — it adds no logic to your website.
