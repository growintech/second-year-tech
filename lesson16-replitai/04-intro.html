Inspect & Download

Find your static files inside the generated project, download them, and run the site locally.

The generated file structure

After the agent runs, the file tree contains more than just your HTML/CSS/JS. Look for the folder with your three static files:

replit-project/
  ├── server.js← keeps Replit preview running (ignore)
  ├── package.json← Node config (ignore)
  └── artifacts/
      └── static-site/
          ├── index.html← your website
          └── public/
              ├── style.css
              └── script.js
💡
The exact folder name varies — look for the folder that contains index.html. That is the one to download.

Download & run locally

  1. In the file tree, right-click the folder that contains index.html (e.g. static-site/).
  2. Select Download — Replit creates a ZIP file and your browser downloads it.
  3. Unzip the archive. You should see index.html and a public/ folder.
  4. Double-click index.html — it opens directly in your browser, no server needed.
  5. Check that the styles load and any JavaScript interactions work.
⚠
If the page loads but styles are missing, check that style.css is linked correctly inside <head> and that the href path matches the actual file location.

The final project

Your final project uses the same delivery format: a ZIP archive containing index.html, style.css, and script.js, uploaded to Classroom.

You may use AI tools to help generate code, but you must be able to read it, explain it, and modify it manually.
