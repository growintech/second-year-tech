Build a website in HTML, CSS and JavaScript. It must have a responsive layout, a working navbar, and clear sections that communicate your startup's problem, solution and product.

You can look at the startup website examples ↗ for inspiration.

Mandatory JavaScript requirements:

Required 1
Launch countdown using Date() and setInterval(). The countdown must update every second and display days, hours, minutes and seconds remaining to a launch date you choose.
Required 2
At least one additional interactive feature that makes sense for your product. Examples: contact form with validation and alert, FAQ accordion, animated counter, embedded map, image slider, newsletter sign-up. 
// Required 1 — Launch countdown
const launchDate = new Date('2025-12-01');

setInterval(function() {
  const diff = launchDate - new Date();
  const days    = Math.floor(diff / (1000 * 60 * 60 * 24));
  const hours   = Math.floor(diff / (1000 * 60 * 60) % 24);
  const minutes = Math.floor(diff / (1000 * 60) % 60);
  const seconds = Math.floor(diff / 1000 % 60);
  // update your DOM elements here...
}, 1000);

// Required 2 — Your choice of interactive feature
🤖 AI is allowed. You can use ChatGPT, Claude, or any other tool to help generate and structure your code. However, you must be able to explain every single line of code during the oral presentation — including anything generated with AI.

2
Prepare the oral presentation
Pair work — before the presentation day
⏱ ~20 min prep

Your presentation must follow this order. Practise it out loud at least once before the day — you will have limited time to rehearse in class.

1
Pitch Deck — walk through your 4 slides: problem, solution, market data, call to action. Keep it under 3 minutes.
2
Website demo — switch to your live site and walk through it. Demonstrate the countdown and the interactive JS feature in real time.
3
Q&A — instructors may ask you to explain specific parts of your code. Both members of the pair must be able to answer.
🎤 Delivery tip. Don't read from slides. Look at the audience. Talk about the problem first — make it feel real before you show the solution. Read the delivery techniques guide ↗ before you practise.
⚡ Deliver on presentation day (in class)
  • Live oral presentation — Pitch Deck + website demo + Q&A
  • Both pair members must present and be able to answer questions about the code
